Learning WordPress

Learning Wordpress Wordpress is one of the worlds largest Content Management Systems(CMS) in use today, and is used in roughly 43% of all websites that are being hosted today. With such a large and important software, it is important to understand just how it is used and why people all over the world are using it to run their websites. During a 16 week long semester at UVU, I was able to learn more about wordpress and host a site managed by wordpress through Amazon Web Services (AWS). By the end of those 16 weeks, I was hoping to be able to better understand how to implement a custom theme into wordpress using SFTP (secure file transfer protocol), learn some of the most popular and useful plugins for wordpress, and ultimately build a site that has all the basic necessities that a fictional business would need to have a basic online store.

Getting Started with AWS

It all started with first understanding AWS and the online hosting. If I couldn't get my code into AWS to be hosted, then there would be no way to get it into wordpress. After looking into what different options were available, I decided to simply use AWS's sub tool S3 buckets. They work by allowing you to directly drop your files into a 'bucket' and then they would use those files to host the site. And so my fake Business Expo Parties was made, and you can still see the very basic home page to this day at http://expogamingparties.s3-website.us-east-2.amazonaws.com/#.

Moving to WordPress with Lightsail

From there I went and learned more about Wordpress. Now that I could get information into AWS it was time to see about linking it to wordpress to host my site. That is where another tool from AWS comes in hand, "Lightsail". Lightsail is a tool offered by AWS that allows users to create a new instance of wordpress, and have it be hosted through AWS. It will even allow you to connect to it using SSH, and SFTP so that as you work in your separate coding interface, you can link it to automatically sync to your site. Using those tools, I quickly moved over from using S3 buckets to Lightsail. We got the SSH key, Username and password and linked out STFT to allow for the site to update our theme automatically. From there we then disabled the 2 default editors that wordpress has to offer and deleted all the preinstalled plugins. With that clean slate, I started working on my theme.

Theme Development

It started by adding the empty theme to the wordpress site and then implementing it on the page. From there I began working in SCSS to build a basic responsive site that had both wayfinding, feedback, and basic functionality. The theme was worked on throughout the entire rest of the project as needed. But when it was to the point where it was sufficient I was able to go in and start creating each page through wordpress.

If I could go back, I would want to refine both of these sites a little more with the further knowledge that I've learned.
